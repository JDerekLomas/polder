How the math works
Active usersprovisioned seats × active %
Resale COGSper user: input tokens × (cache-adjusted, routing-blended in-price) + output tokens × routing-blended out-price; × active users
Self-host COGSsize by output tokens; cheap-routed fraction uses ~0.2× GPU; nodes = ⌈effective output ÷ node capacity⌉; × node cost; minus batch backfill offset. Whole nodes only — small scale pays for a full node.
Chosen COGSmin(resale, self-host) — the crossover is where owning a node beats reselling
Netrevenue − chosen COGS − (team × loaded ÷ 12) − compliance ÷ 12

A deliberately transparent model, not a guarantee. The softest knobs — node throughput, utilization, active ratio, and tokens/user — swing the result most; instrument them in a real pilot before trusting any single output. One-time costs (ISO setup ~€55k, build engineering, sales) and ramp are excluded. USD figures converted at ~0.92.
